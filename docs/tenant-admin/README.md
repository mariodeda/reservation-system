# Tenant Admin Guide

Tenant admin lives at `/admin/<slug>`. It is for restaurant staff managing daily
operations.

## Main Areas

- Dashboard: today's reservations and fast operational actions.
- Reservations: calendar, list, floor/day modal, waitlist modal, reservation
  actions, table assignment, and review request actions.
- Tables: dining-room capacity and table metadata.
- Availability: weekly hours, special dates, closed days, blocked slots, lead
  time, booking window, party-size policy, and service durations.
- Clients & Statistics: customer records and analytics.
- Settings: tenant-owned local settings and staff password change.

## Tenant Authority

The session determines tenant access. A staff user cannot select another tenant
by changing the URL slug.

## Daily Workflow

1. Use the dashboard for today's reservations.
2. Use quick booking controls to stop today's remaining online bookings for a
   service when needed.
3. Use reservations to manage guest status, table assignment, walk-ins, waitlist,
   and review requests.
4. Use availability when schedule or policy changes are needed.
