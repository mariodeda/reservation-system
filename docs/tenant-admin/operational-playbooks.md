# Operational Playbooks

This page gives staff step-by-step responses to common real-world situations.
Use it as a practical service guide.

## Stop Online Bookings For Today

Use when the restaurant can no longer accept more online bookings for a service.

Steps:

1. Open the header quick action.
2. Find today's affected service.
3. Toggle the service off.
4. Confirm the service appears stopped.
5. Check Reservations slot cards for stopped-service messaging.
6. Tell staff that existing reservations remain valid.

Remember: stopping bookings does not cancel existing reservations.

## Reopen A Service Today

Use only if the restaurant can accept more online bookings again.

Steps:

1. Open the header quick action.
2. Find the stopped service.
3. Toggle it back on if the control is still available.
4. Check slot cards for availability.
5. Confirm that the latest bookable time has not already passed.

If the switch is disabled, the service may already be past the latest bookable
time.

## Handle A Phone Booking

1. Open Reservations.
2. Select the correct date and bookable area.
3. Choose the requested service and time.
4. Click the slot or open the new reservation modal.
5. Enter guest name, party size, phone, email, and notes.
6. Save.
7. Assign a table if appropriate.
8. Repeat the details back to the guest.

If the guest has no email, make sure the phone number is accurate.

## Handle A Walk-In

1. Check current slot and table state.
2. Add a staff reservation or waitlist entry depending on capacity.
3. If seating immediately, assign a table and mark seated.
4. Add notes if the party has a time limit or special need.
5. Mark completed after the visit.

## Handle A Large Party

1. Check the maximum party size rule.
2. Check how many seats are available in the slot.
3. Check table assignment possibilities.
4. Use joined tables only if physically realistic.
5. Add staff notes for preparation.
6. Confirm arrival time and contact details by phone.

If online booking blocks the party but the restaurant wants to accept it
manually, staff may create it as a manager-approved exception if table conflicts
can be handled.

## Handle A Guest Running Late

1. Keep the reservation visible.
2. Add or update notes if available.
3. Check floor/day modal for downstream conflicts.
4. Adjust table assignment only if needed.
5. Do not mark no-show until the restaurant's grace period passes.

## Handle A No-Show

1. Wait until the restaurant's normal grace period has passed.
2. Mark the reservation no-show.
3. Do not send review request email.
4. Use analytics later to track repeated no-show patterns.

## Handle A Cancellation

1. Open the reservation.
2. Mark cancelled rather than deleting.
3. Keep useful cancellation notes.
4. Confirm whether the slot should now be available again.
5. Do not send review request email.

## Handle A Guest Email Warning

1. Treat the warning as a request for staff follow-up.
2. Call the guest using the phone number on the reservation.
3. Confirm the booking details verbally.
4. Ask for a corrected email if needed.
5. Update contact details where possible.

Email warnings can come from an immediate rejection or a later delivery failure.
Either way, do not rely on email for that guest until corrected.

## Handle "Could Not Load Availability"

1. Confirm internet connection.
2. Refresh the page.
3. Confirm the staff session has not expired.
4. Try another date.
5. Check whether availability settings were recently edited.
6. Ask platform support to check logs if the issue persists.

Do not keep creating manual bookings blindly if availability is failing. The
restaurant may lose conflict protection.

## Handle Suspicious Duplicate Notifications

1. Check whether multiple browser tabs are open.
2. Compare guest name, time, date, and party size.
3. Confirm whether there is one reservation or multiple reservations.
4. Mark notifications read.
5. Refresh the active tab.
6. Escalate if the same reservation keeps producing new unread notifications.

Duplicate notifications are different from duplicate reservations. Always check
the reservation list before calling the guest.

## End-Of-Day Closeout

1. Mark all attended reservations completed.
2. Mark no-shows accurately.
3. Confirm cancellations are not left as confirmed.
4. Review any email warnings and follow-up notes.
5. Send review requests for completed reservations when appropriate.
6. Check tomorrow's large parties and special notes.
7. Sign out on shared devices.
